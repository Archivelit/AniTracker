# Contributing

Thanks for getting interested in contributing this project. Just follow rules below to make contributing easy for everyone. 

## How to contribute

### Reporting bugs

To report bug, just create an issue. In issue you should: 
- Ensure that issue doesn't exists (simply upvote issue if it does)
- Describe bug
- Suggest the solution

That's it.

### Opening a pull request

- Create a fork of this project
- Create new branch with appropriate name
- Run formatter using `dotnet format`
- Ensure your code passes all tests and github actions workflows
- Describe what changes introduce your PR 

### Commit messages

Use clear and focused commit messages to describe changes introduced by commit (e.g. "Fix crash produced by data validation #12345", "Add regex support")

### Documentation 

Any code introduced in PR must be documented.
